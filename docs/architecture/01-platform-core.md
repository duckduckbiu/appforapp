# 01 — 愿景与三层架构

> 本文件属于 [Bill.ai 平台架构设计文档](../PLATFORM_ARCHITECTURE.md) 的一部分。

---

## 1.1 一句话定位

**Bill.ai = AI 时代的微信小程序平台**

用户安装一个 APP，通过 AI 或开发者生成无限定制应用。

## 1.2 五层角色体系

| 角色 | 说明 | 类比 | 如何获得 |
|------|------|------|---------|
| **平台** | Bill.ai 本身，提供基础设施 | 微信 / Discord / App Store | — |
| **开发者** | 创建和发布应用 | 小程序开发者 | 创建第一个应用时自动解锁 |
| **频道主** | 运营社区/频道，安装应用到频道 | Discord 服务器主 | 创建第一个频道时自动解锁 |
| **推广者** | 推广应用/频道，拉新用户 | 淘宝客 / 抖音达人 | 生成第一个推广链接时自动解锁 |
| **用户** | 使用应用，消费内容 | 普通用户 | 注册即是 |

> **能力解锁模型**：所有人注册进来都是「用户」。角色不是注册时选择的身份，而是做了某件事后自动获得的能力。一个人可以同时拥有所有角色。

## 1.3 法律架构

```
Bill.ai 平台（中立基础设施）
    ↓ 提供 SDK + 沙盒 + 应用商店
第三方开发者 发布应用（如 Rich Game）
    ↓
频道主 选择安装到自己的频道
    ↓
推广者 推广频道/应用
    ↓
用户自行选择安装和使用

平台免责 (Safe Harbor):
  - Bill.ai 是工具提供者，不直接运营具体应用
  - 类似 Discord 不为服务器中的内容负责
  - 类似 App Store 不为应用内容负责
```

## 1.4 三层架构

```
┌───────────────────────────────────────────────────────┐
│                   核心层 (Core Layer)                   │
│  认证 · 路由 · 布局 · 结算 · 安全 · 审核              │
│  → 高稳定性，不可替换，平台团队维护                      │
├───────────────────────────────────────────────────────┤
│               平台功能层 (Platform Features)            │
│  用户中心 · 社交 · 聊天 · Feed · 钱包 · 通知 · 应用商店  │
│  → 模块化，可扩展，通过 App SDK 对外暴露                 │
├───────────────────────────────────────────────────────┤
│                  应用层 (Apps Layer)                    │
│  Rich Game · AI 生成应用 · 第三方应用 · ...              │
│  → 沙盒化，数据隔离，热插拔                              │
└───────────────────────────────────────────────────────┘
```

### 核心层职责

| 模块 | 职责 | 技术 |
|------|------|------|
| 认证 | 邮箱/Google/Apple 登录，JWT 签发 | Supabase Auth |
| 路由 | 平台级页面路由 + 应用沙盒加载 | React Router |
| 布局 | 顶栏 + 侧栏 + 内容区（三段式） | React + CSS |
| 结算 | 平台钱包、应用分账、IAP 接入 | Supabase + Stripe |
| 安全 | CSP、沙盒隔离、API 鉴权 | iframe sandbox |
| 审核 | 应用审核、内容审核、年龄分级 | AI + 人工 |

### 平台功能层（通过 SDK 暴露给应用）

| 模块 | 对应 SDK 命名空间 | 说明 |
|------|-------------------|------|
| 用户中心 | `billai.user` | 用户资料、设置 |
| 社交关系 | `billai.social` | 好友、关注、社交图谱 |
| 聊天消息 | `billai.chat` | 私聊、群聊（平台统一管理） |
| Feed/动态 | `billai.feed` | 发帖、评论、点赞、分享 |
| 钱包 | `billai.wallet` | 余额查询、充值、扣款、转账 |
| 通知 | `billai.notification` | 灵动岛、系统推送 |
| 应用商店 | `billai.store` | 应用搜索、安装、卸载 |
| 权限 | `billai.permission` | 申请/查询权限 |
| UI | `billai.ui` | toast、弹窗、主题色 |
| 订阅 | `billai.subscription` | 订阅管理、付费墙 |
| 存储 | `billai.storage` | 文件上传、下载、管理 |

### 应用层（沙盒内运行）

每个应用：
- 运行在 **iframe** 中（Web）或 **WebView** 中（移动端）
- 有独立的前端代码（HTML/JS/CSS）
- 可选有独立的后端服务（如 Rich Game 的 Express 后端）
- 通过 `postMessage` 与平台通信
- 不能直接访问平台数据库、DOM、Cookie
